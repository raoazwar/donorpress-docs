---
title: "Webhooks"
description: "How DonorPress hears about events that happen outside your site."
---

A **webhook** is a notification the gateway sends to your site when something happens — a subscription renewal, a refund processed in the gateway dashboard, a chargeback, etc. Without webhooks, DonorPress has no way to know about these events.

<Warning>
	**Always configure webhooks for production sites.** Without them, recurring donations will technically still process at the gateway, but DonorPress won't know about them — your reports will be wrong and donors won't get renewal receipts.
</Warning>

## Webhook endpoints

DonorPress exposes one webhook endpoint per gateway:

| Gateway | Endpoint URL |
| --- | --- |
| Stripe | `https://yoursite.com/wp-json/donorpress/v2/gateways/stripe/webhook` |
| PayPal | `https://yoursite.com/wp-json/donorpress/v2/gateways/paypal/webhook` |

Replace `yoursite.com` with your actual domain.

## What webhooks DonorPress listens for

### Stripe

| Event | What DonorPress does |
| --- | --- |
| `payment_intent.succeeded` | Marks the donation as Complete. |
| `payment_intent.payment_failed` | Marks the donation as Failed. |
| `charge.refunded` | Marks the donation as Refunded (or partial). |
| `customer.subscription.created` | Records the new subscription. |
| `customer.subscription.updated` | Reflects status / price changes. |
| `customer.subscription.deleted` | Marks the subscription as Cancelled. |
| `invoice.payment_succeeded` | Records the renewal donation. |
| `invoice.payment_failed` | Marks the subscription as Failing. |

### PayPal

| Event | What DonorPress does |
| --- | --- |
| `CHECKOUT.ORDER.APPROVED` | Captures the order. |
| `CHECKOUT.ORDER.COMPLETED` | Marks the donation as Complete. |
| `PAYMENT.CAPTURE.COMPLETED` | Same as above, for redundancy. |
| `PAYMENT.CAPTURE.REFUNDED` | Marks the donation as Refunded. |
| `PAYMENT.CAPTURE.DENIED` | Marks the donation as Failed. |
| `BILLING.SUBSCRIPTION.ACTIVATED` | Records the subscription as Active. |
| `BILLING.SUBSCRIPTION.CANCELLED` | Marks the subscription as Cancelled. |
| `BILLING.SUBSCRIPTION.SUSPENDED` | Marks the subscription as Paused. |
| `BILLING.SUBSCRIPTION.PAYMENT.FAILED` | Marks the subscription as Failing. |

## Test vs. live webhooks

Each gateway has separate test and live webhooks:

- **Test webhooks** point at the same DonorPress endpoint and use your test signing secret.
- **Live webhooks** point at the same endpoint but use your live signing secret.

DonorPress automatically picks the right secret based on which credentials processed the underlying payment.

## Signature verification

Every webhook request is verified against the gateway's signing secret:

- **Stripe** — HMAC-SHA256 over the request body, compared to the `Stripe-Signature` header.
- **PayPal** — RSA-SHA256 verification using PayPal's public certificate, compared to the `PAYPAL-TRANSMISSION-SIG` header.

If verification fails, DonorPress responds with HTTP 401 and logs the failed attempt. This prevents spoofed webhooks from creating fake donations.

## Local development

For local sites that aren't reachable from the internet, use a tunnel:

- [ngrok](https://ngrok.com/) — `ngrok http 80` gives you a public URL pointing at your local site.
- [Cloudflare Tunnel](https://www.cloudflare.com/products/tunnel/) — alternative, free.
- [Localtunnel](https://localtunnel.github.io/www/) — simpler ngrok alternative.

Configure the gateway to send test webhooks to your tunnel URL.

## Inspecting webhook activity

In **DonorPress → Tools → Webhook Log** you'll find every webhook delivery (signature verified, body parsed, action taken, response code returned). Useful when debugging "why didn't this renewal show up?"

You can also retry a webhook from the log if it failed for a transient reason.

## Common issues

<AccordionGroup>
	<Accordion title="401 Unauthorized">
		The signing secret in DonorPress doesn't match what the gateway is signing with. Re-copy from the gateway dashboard.
	</Accordion>
	<Accordion title="404 Not Found">
		Your site's permalinks aren't flushed. Go to **Settings → Permalinks** and click Save (no changes needed) to flush.
	</Accordion>
	<Accordion title="500 errors during webhook processing">
		Usually a PHP error. Check your WordPress debug log (`wp-content/debug.log`) for the stack trace.
	</Accordion>
	<Accordion title="Webhook events appear delayed">
		Some gateways batch webhook deliveries. If you see a delay > 30 seconds, check the gateway's status page.
	</Accordion>
	<Accordion title="Webhooks work in test but not in live">
		You probably configured only the test webhook in the gateway. Live needs its own endpoint with the live signing secret.
	</Accordion>
</AccordionGroup>
