The Subscriptions admin (**DonorPress → Subscriptions**) is where you manage every recurring donation on your site.

## List view

| Column | Description |
| --- | --- |
| ID | Internal subscription ID. |
| Donor | Donor name + email. |
| Amount | Per-renewal amount. |
| Frequency | Daily, Weekly, Monthly, Quarterly, or Yearly. |
| Status | Active, Paused, Cancelled, Failing, Expired, Completed. |
| Next renewal | Date of the next scheduled charge. |
| Started | Date the subscription was created. |

## Filters

- **Status** — show only active, paused, etc.
- **Gateway** — Stripe / PayPal / etc.
- **Frequency**
- **Date range** — when the subscription started.
- **Donor** — by name or email.

## Inspecting a subscription

Click any row to open the detail screen:

- **Summary** — amount, frequency, next renewal, gateway info.
- **Donor card** — link to the donor record.
- **Renewals** — full list of every donation generated by this subscription.
- **Activity log** — every event (created, renewed, paused, gateway response, etc.).
- **Notes** — internal admin notes.

From here you can pause, cancel, or skip the next renewal.

## Pausing

A paused subscription:

- Does not process the next scheduled renewal.
- Remains in the gateway in a paused/skipped state.
- Can be resumed by the admin from the same screen.
- Donor receives a "Subscription Paused" email (if the template is configured).

Use pause for temporary holds (donor going on extended travel, etc.). For permanent stops, cancel instead.

## Cancelling

A cancelled subscription:

- Stops all future renewals immediately.
- Sends a cancellation request to the gateway (so PayPal/Stripe stop too).
- Cannot be reactivated. To restart, create a new subscription.
- Donor receives a "Subscription Cancelled" email.

## Skipping the next renewal

For one-off skip without cancelling:

1. Open the subscription detail.
2. Click **Skip next renewal**.
3. Confirm.

The next charge is bypassed; subsequent renewals continue on schedule. Useful for end-of-year breaks or one-off financial hardship requests.

## Failing renewals

When a renewal fails, the subscription enters **Failing** status. The gateway will retry per its own dunning rules:

- **Stripe** — typically 3 retries over a week, then cancels the subscription.
- **PayPal** — similar, configurable in your PayPal account settings.

DonorPress sends the donor a "Renewal Failed" email after the first failure, asking them to update payment details. After the gateway gives up, the subscription moves to **Cancelled**.

## Manually retrying a failed renewal

From the subscription detail, click **Retry renewal**. This forces the gateway to attempt the next charge immediately. Useful when the donor confirms they've fixed their card.

## Updating payment method

DonorPress doesn't currently expose a donor-facing payment-method-update flow. To update a donor's card on a Stripe subscription:

1. The donor opens their Stripe billing portal (you can email them a link from the subscription detail screen).
2. They update the card.
3. The next renewal uses the new card.

For PayPal, the donor must cancel and create a new subscription with the updated funding source.

## Bulk actions

- **Pause** — pauses all selected subscriptions.
- **Cancel** — cancels all selected. Use with care.
- **Export** — CSV of all selected subscriptions.
