---
title: "Subscriptions Endpoint"
description: "REST API: list, pause, cancel, and inspect recurring donations."
---

Base path: `/wp-json/donorpress/v2/subscriptions`

## List subscriptions

```http
GET /subscriptions
```

### Query parameters

| Param | Type | Description |
| --- | --- | --- |
| `per_page` | int | 1-100. Default 20. |
| `page` | int | Default 1. |
| `status` | string | Comma-separated: `active,paused,cancelled,failing,expired,completed`. |
| `gateway` | string | Comma-separated gateway slugs. |
| `donor_id` | int | Filter by donor. |
| `frequency` | string | `daily,weekly,monthly,quarterly,yearly`. |
| `from` | date | Started after this date. |
| `to` | date | Started before this date. |
| `orderby` | string | `created_at` / `renewal_date` / `amount`. |
| `order` | string | `ASC` / `DESC`. |

### Response

```json
[
  {
    "id": 17,
    "donor_id": 7,
    "donor": { "id": 7, "email": "sam@example.com", "first_name": "Sam" },
    "form_id": 3,
    "campaign_id": 0,
    "amount": 25.00,
    "currency": "USD",
    "period": "monthly",
    "bill_count": 0,
    "bills_processed": 4,
    "status": "active",
    "gateway": "stripe",
    "gateway_subscription_id": "sub_1OxYzAB...",

    "updated_at": "2025-04-22T14:38:14Z",
    "cancelled_at": null
  }
]
```

## Get a single subscription

```http
GET /subscriptions/{id}
```

Returns the same shape, plus optional expansions:

- `?include=renewals` — every donation generated by this subscription.
- `?include=activity` — full activity log.

## Pause a subscription

```http
POST /subscriptions/{id}/pause
```

### Response

```json
{
  "id": 17,
  "status": "paused",
  ...
}
```

The next renewal is skipped. The subscription remains paused until you resume it.

## Resume a subscription

```http
POST /subscriptions/{id}/resume
```

Returns the subscription with status back to `active` and an updated `renewal_date`.

## Skip the next renewal

```http
POST /subscriptions/{id}/skip
```

Skips one renewal without changing the overall status. Useful for one-off skips (donor going on extended travel, etc.). The subscription stays Active.

## Cancel a subscription

```http
POST /subscriptions/{id}/cancel
```

### Body (optional)

```json
{
  "reason": "Donor requested",
  "cancel_at_period_end": false
}
```

| Field | Description |
| --- | --- |
| `reason` | Optional free text. Stored in activity log. |
| `cancel_at_period_end` | If `true`, lets the donor finish out the current paid period. Stripe only. |

### Response

```json
{
  "id": 17,
  "status": "cancelled",
  "cancelled_at": "2025-04-22T15:00:00Z",
  ...
}
```

## Manually retry a failed renewal

```http
POST /subscriptions/{id}/retry
```

Forces the gateway to attempt the next charge immediately. Useful when the donor confirms they've fixed their card.

## Update a subscription

```http
PATCH /subscriptions/{id}
```

Limited editable fields:

```json
{
  "campaign_id": 12,
  "note": "Customer requested attribution to spring campaign"
}
```

You **cannot** change the amount, frequency, or gateway of an existing subscription via this endpoint. Stripe and PayPal both require the donor to cancel and create a new subscription for those changes.
