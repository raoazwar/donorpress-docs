---
title: "Reports Endpoint"
description: "REST API: revenue, donor, form, and subscription analytics."
---

Base path: `/wp-json/donorpress/v2/reports`

Reports endpoints aggregate raw donation / donor data into the same numbers shown in the admin reports UI.

## Common parameters

All report endpoints accept a date range:

| Param | Type | Description |
| --- | --- | --- |
| `from` | date | YYYY-MM-DD. Inclusive. |
| `to` | date | YYYY-MM-DD. Inclusive. |
| `period` | string | Shorthand: `last_7d`, `last_30d`, `this_month`, `last_month`, `this_year`, `last_year`. Mutually exclusive with `from` / `to`. |
| `gateway` | string | Comma-separated gateway slugs. |
| `currency` | string | ISO 4217 code. Defaults to site currency. |
| `compare_previous` | bool | If true, return data for the previous equivalent period in a `previous` key. |

## Custom report builder

```http
POST /reports/custom
```

Build ad-hoc aggregations without writing SQL.

### Body

```json
{
  "metric": "sum_amount",       // sum_amount, count, avg_amount, count_distinct_donors
  "group_by": "gateway",        // gateway, form, campaign, day, week, month, quarter, year, status
  "filters": {
    "status": ["complete"],
    "gateway": ["stripe"],
    "from": "2025-01-01",
    "to": "2025-04-30"
  }
}
```

### Response

```json
{
  "rows": [
    { "group": "stripe", "value": 9210.00 }
  ]
}
```

Powerful for custom dashboards. Use sparingly — large unfiltered queries can be slow.

## Permissions

All endpoints require `manage_options` (or `donorpress_view_reports` if you've configured a separate reports capability).
